# Validation Rules

## Parameter Validation Rules

- `surahId`: 1-114 (ada 114 Surah dalam Al-Qur'an)
- `ayahNumber`: tergantung Surah, minimal 1
- `page`: minimal 1
- `limit`: 1-100 untuk pagination
- `q`: 1-100 karakter untuk search

## Validation Library

API menggunakan Zod schema validation untuk semua parameter incoming.

## Error Response untuk Validation

Ketika validation gagal, API akan mengembalikan error 400 dengan struktur:

```json
{
  "error": {
    "issues": [
      {
        "code": "too_small",
        "minimum": 1,
        "type": "number",
        "inclusive": true,
        "exact": false,
        "message": "Number must be greater than or equal to 1",
        "path": ["page"]
      }
    ],
    "name": "ZodError"
  }
}
```

## Rules Detail

### Surah ID

- Valid range: 1 sampai 114
- Tipe: integer
- Wajib ada untuk endpoint Surah detail

### Ayah Number

- Minimal: 1
- Maksimal: tergantung Surah (misal Al-Fatihah punya 7 ayat)
- Validasi dikonfirmasi terhadap data di database

### Pagination

- `page`: integer >=1
- `limit`: integer antara 1-100

### Search Query

- Minimal 1 karakter
- Maksimal 100 karakter
- String non-empty
